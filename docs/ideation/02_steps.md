# 프로젝트 단계별 실행 계획

## Phase 1: 프로젝트 환경 설정
1.  **Git 저장소 설정**: 프로젝트를 위한 Git 저장소를 초기화하고 기본 구조를 설정합니다.
2.  **Kubernetes 클러스터 구축**: 마이크로서비스를 배포하고 관리하기 위한 Kubernetes 클러스터를 설정합니다.
3.  **DNS (Bind9) 설정**: 서비스 검색 및 라우팅을 위해 DNS 서버를 구성합니다.
4.  **NFS 설정**: 영구 데이터를 저장하기 위한 NFS(Network File System)를 설정합니다.

## Phase 2: 데이터베이스 설정
1.  **MySQL 스키마 정의**: `01_brain_stroming.md`에 정의된 테이블 구조를 기반으로 MySQL 데이터베이스 스키마를 작성합니다.
2.  **데이터베이스 초기화 스크립트 작성**: 데이터베이스와 테이블을 생성하고 초기 데이터를 삽입하는 스크립트를 작성합니다.
3.  **MySQL 배포**: Kubernetes 클러스터에 Master-Slave 구조의 MySQL을 배포합니다.

## Phase 3: 백엔드 개발 (FastAPI)
1.  **통합 백엔드 서비스 개발**: 초기에는 단일 FastAPI 애플리케이션으로 모든 백엔드 로직을 구현합니다.
2.  **인증 및 권한 부여 구현**: JWT 기반의 인증 및 역할 기반 권한 부여 시스템을 구현합니다.
3.  **API 엔드포인트 개발**:
    *   관리자 기능 (회원, 업체, 상품, 주문 관리)
    *   입점 업체 관리자 기능 (업체 정보, 상품, 주문 관리)
    *   고객 기능 (상품 조회/검색, 주문, 리뷰 작성)
4.  **컨테이너화**: FastAPI 애플리케이션을 Docker 이미지로 빌드합니다.
5.  **백엔드 배포**: 빌드된 Docker 이미지를 Kubernetes 클러스터에 배포합니다.

## Phase 4: 프론트엔드 개발
1.  **프론트엔드 애플리케이션 개발**:
    *   관리자 페이지
    *   입점 업체 관리자 페이지
    *   고객 페이지
2.  **컨테이너화**: Nginx를 사용하여 정적 파일을 서비스하는 Docker 이미지를 빌드합니다.
3.  **프론트엔드 배포**: 빌드된 Docker 이미지를 Kubernetes 클러스터에 배포합니다.

## Phase 5: 통합 및 배포
1.  **Ingress Controller 설정**: 외부 트래픽을 프론트엔드 및 백엔드 서비스로 라우팅하도록 Ingress Controller를 구성합니다.
2.  **전체 시스템 테스트**: 회원가입부터 주문까지 전체 애플리케이션 흐름을 테스트합니다.
3.  **(선택 사항) 모니터링 및 로깅**: Prometheus, Grafana, ELK 스택 등을 사용하여 모니터링 및 로깅 시스템을 구축합니다.
